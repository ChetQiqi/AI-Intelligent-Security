# 安防驾驶舱使用说明

## 功能概览

安防驾驶舱用于展示实时安防状态、最近事件、门禁异常、人脸识别结果、摄像头运行状态、NL2SQL 查数和 RAG 知识库问答。系统目标是帮助值班人员快速发现异常、定位原因并完成处置记录。

## 驾驶舱总览

驾驶舱总览展示关键指标，包括今日事件数、识别准确率、在线摄像头、关键风险事件、今日已识别人员和系统状态。值班员应优先查看红色或橙色风险指标。

## 最近事件

最近事件列表展示最新的识别事件、门禁事件和系统告警。每条事件通常包含：

- 时间。
- 地点。
- 摄像头或门禁点。
- 人员名称或陌生人标记。
- 置信度。
- 事件等级。

如果最近事件中同一地点连续出现异常，应进入 NL2SQL 页面进一步查询。

## NL2SQL 查数

NL2SQL 页面支持用自然语言查询数据库。示例问题：

- 最近 1 小时有哪些陌生人员识别事件？
- 今天拒绝访问次数最高的门禁点是哪里？
- 查询摄像头 CAM-01 最近 30 分钟的识别记录。
- 统计本周每个区域的异常事件数量。

系统会生成 SQL、执行查询并展示图表、表格和 AI 分析结论。执行关键查询前，应检查生成 SQL 是否符合预期。

## RAG 知识库

RAG 知识库用于上传制度、SOP、运维手册和历史事件复盘。值班员可以用自然语言提问，例如：

- 限制区域出现陌生人应该怎么处理？
- 摄像头离线超过 10 分钟应该通知谁？
- 夜间门禁拒绝访问如何分级？
- 人脸识别置信度低可能是什么原因？

回答中的引用来源用于追溯具体文档。若答案与现场情况冲突，应以现场确认和主管决策为准。

## 推荐工作流

1. 先在驾驶舱总览确认风险等级。
2. 查看最近事件，判断是否连续异常。
3. 使用 NL2SQL 查询相关记录。
4. 使用 RAG 检索处置规范。
5. 完成现场确认。
6. 记录处置结论和误报原因。

## 使用注意事项

- 不要只凭单次低置信度识别判断人员异常。
- 陌生人事件需要结合门禁记录和现场视频确认。
- 系统建议不是最终处置命令。
- 涉及限制区域、黑名单人员或摄像头离线的事件应优先处理。
