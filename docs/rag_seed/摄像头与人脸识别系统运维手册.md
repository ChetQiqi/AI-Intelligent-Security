# 摄像头与人脸识别系统运维手册

## 系统组成

人脸识别系统由摄像头、视频流处理服务、人脸检测模型、人脸识别模型、特征库、事件数据库和安防驾驶舱组成。系统会对摄像头画面进行实时检测，并将识别结果、置信度、延迟和事件信息写入数据库。

## 摄像头状态

摄像头状态分为在线、离线、延迟高、画面异常和维护中。

在线表示摄像头可以正常拉流并返回画面。离线表示无法连接或连续多次拉流失败。延迟高表示画面可用但处理延迟超过阈值。画面异常包括遮挡、黑屏、强光、模糊和角度偏移。

## 常见故障排查

### 摄像头离线

处理步骤：

1. 检查摄像头网络是否可达。
2. 检查 RTSP 地址、账号和密码是否变更。
3. 查看同区域其他摄像头是否同时离线。
4. 检查交换机、电源和网络带宽。
5. 若 10 分钟内无法恢复，应通知运维人员现场处理。

### 识别置信度降低

识别置信度降低通常由光照、角度、遮挡、运动模糊或底库照片过旧导致。处理方式：

- 优先查看摄像头画面质量。
- 检查是否存在逆光或夜间补光不足。
- 确认人员是否佩戴口罩、帽子或护目镜。
- 对长期低置信度人员更新底库照片。

### FPS 下降

FPS 下降可能由模型推理速度、摄像头数量、GPU 占用或视频分辨率导致。排查重点：

- GPU 显存和利用率。
- 当前检测模型是否为 MTCNN。
- 视频分辨率是否过高。
- 是否同时运行 AI 肖像生成任务。

当前 MTCNN 检测器准确但速度较慢，如目标是 25 FPS 以上，可考虑替换为 YOLOv8-Face。

## 性能指标

系统应关注以下指标：

- FPS：每秒处理帧数。
- latency_ms：单帧处理延迟。
- recognition_confidence：识别置信度。
- false_accept_rate：误接受率。
- false_reject_rate：误拒绝率。
- unknown_detection_rate：陌生人检测率。

## 维护建议

- 每周检查摄像头在线率。
- 每月更新重点人员底库照片。
- 每次算法阈值调整后重新跑评估集。
- 重大事件后导出 CSV/JSON 指标作为复盘材料。
- 避免在高峰识别时段运行高显存任务。
