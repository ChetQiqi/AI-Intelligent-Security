### 1.4 识别推理

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/recognize` | 单张图片人脸识别 | 全部 |
| POST | `/recognize/video` | 视频逐帧识别 + 跟踪平滑 | 全部 |

```python
# POST /recognize (multipart)
Request:  file (UploadFile) + threshold (float, 默认 0.45)
Response: {
    "results": [{
        "box": [x, y, w, h],     # 人脸框
        "name": str,              # 匹配到的身份名
        "display_name": str,      # 显示名称（含 Unknown）

        "raw_score": float,       # 原始 cosine 相似度
        "accepted": bool,         # 是否超过阈值
        "label": str,             # "Accepted" | "Rejected"
        "support": int            # 图库中该身份的特征数
    }, ...],
    "annotated_image_base64": str | null,  # 标注后的图片（base64 JPEG）
    "matched_pairs": [{
        "name": str,
        "gallery_face_base64": str | null,  # 图库参考人脸
        "query_image_base64": str | null,   # 被识别的原图（含框）
        "gallery_error": str | null         # 图库图片获取失败原因
    }, ...]
}

# POST /recognize/video (multipart)
Request:  file (UploadFile) + skip_frames (int, 默认 5) + threshold (float, 默认 0.45)
          + stable_frames (int, 默认 3) + mode (str, "视频中找人"|"验证ID")
          + verify_target (str, 可选)
Response: {
    "video_base64": str,          # H.264 编码的结果视频
    "output_filename": str,       # 输出文件名
    "total_frames": int,          # 视频总帧数
    "processed_frames": int,      # 实际处理的帧数
    "written_frames": int,        # 写入输出视频的帧数
    "fps": int,                   # 视频原始帧率
    "width": int, "height": int,  # 视频分辨率
    "person_counts": {str: int},  # 每个识别到的人出现的次数
    "stranger_count": int,        # 未注册人脸次数
    "verify_result": {            # 仅验证模式
        "passed": bool,
        "target": str,
```

## 4. 服务层架构

```
API 层 (api/main.py)
  ├── AuthService          → 认证/注册/用户管理
  ├── IdentityService      → 人员 CRUD + 人脸录入
  │     └── ModelService   → 加载模型/检测器/gallery
  ├── InferenceService     → 图片/单帧识别 + 匹配对构建
  │     └── ModelService
  ├── VideoService         → 视频处理 + 摄像头管理
  │     ├── InferenceService
  │     ├── FaceTracker    → 时序跟踪平滑
  │     └── CameraThread   → 实时摄像头守護線程
  ├── DeveloperService     → Benchmark + Gallery 分析
  ├── ModelManagementService → 模型上传/激活/删除
  │     ├── ModelRegistry  → 激活模型状态管理（单例）
  │     └── ModelRepository → SQLite 模型元数据
  └── EvalService          → 评估任务队列 + 子进程管理
        └── ModelService
```

**核心识别流水线** (`core/operations.py`):
```
输入帧 (BGR)
  → FaceDetector (MTCNN) → 人脸框列表
  → 每人脸: crop → FaceEmbeddingModel (iResNet) → 512-d embedding
  → Matcher: cosine_similarity + topk_mean 聚合
  → similarity_con() 校准
  → 阈值比较 → Accepted / Rejected
  → [可选] adaptive_recognize_face() 三层决策:
     1. z-score 离群检测 (>3.0)
     2. 自适应阈值检查
     3. 距离比不确定性门控 (<1.2)
```

## 6. 识别流水线

### 6.1 模型支持
- **骨干网络**: iResNet-18/34/50/100（PyTorch）
- **权重格式**: `.pt`, `.pth`, `.ckpt`, `.bin`（自动兼容多种 checkpoint 格式）
- **ONNX 支持**: 可选，通过 `OnnxEmbeddingModel`
- **检测器**: MTCNN（TensorFlow 后端，写死）

### 6.2 图库模式
- **mean**: 每人的所有 embedding 取均值作为原型向量
- **all**: 保留每个人的所有 embedding（更精确但更慢）

### 6.3 分数校准
`similarity_con(score)` — 分段线性映射函数将原始 cosine 相似度映射到 [0, 1]

### 6.4 时序跟踪 (`core/tracker.py`)
- **FaceTracker**: 基于 IoU 的多目标跟踪
- 配置: history_size=5, min_stable_count=3, iou_threshold=0.3
- 输出: 稳定识别结果（需连续 N 帧一致）

### 6.5 自适应阈值 (`core/adaptive_threshold.py`)
- 为每个注册人员单独计算阈值（mean - k*std，k=2.0）
- 三层决策: z-score → 自适应阈值 → 距离比
- 支持在线 EMA 更新（learning_rate=0.1）

## 7. 评估任务系统

### 7.1 任务生命周期
```
POST /developer/eval/lfw (提交)
  → 创建 EvalJob (status="pending")
  → 启动子进程 (python tools/eval_lfw_runner.py)
  → status → "running"
  → 子进程 stdout 输出 __PROGRESS__ JSON 行
  → 解析进度 (progress 0-100 + progress_data)
  → 子进程退出 → 解析最终结果
  → status → "done" (或 "error")

前端轮询: GET /developer/eval/status/{job_id} (每 2 秒)
取消: POST /developer/eval/cancel/{job_id} (终止子进程)
```

### 7.2 评估类型详情

| 类型 | 运行器 | 指标 |
|------|--------|------|
| LFW 验证 | `tools/eval_lfw_runner.py` | 6 数据集准确率 + 平均准确率 |
| IJB 评估 | `tools/eval_ijb_runner.py` | TAR@FAR(1e-6~1e-1) + ROC AUC |
| 阈值扫描 | `core/eval_comprehensive.py` | Rank-1/FAR/FRR/F1@多阈值 |
